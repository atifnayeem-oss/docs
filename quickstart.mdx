---
title: "Quickstart"
description: "See recognition and a Checkout Session work in test mode in a minute, on any platform, then add Ringup to your agent."
---

Ringup adds identity and payments to a voice agent: a returning caller is recognized by phone
number and pays with a saved card, settling on the merchant's own processor, with no card number
ever spoken. This quickstart shows the two moves everything is built on, recognize a caller and
charge their saved card, in test mode, then points you to the integration for your platform.

Test mode runs on a shared sandbox with ready-made merchants and callers, so there is nothing to
sign up for and nothing to connect between you and a first working response. See
[Testing](/concepts/testing) for the fixtures.

<Steps>
  <Step title="Recognize a caller">
    Ringup recognizes a caller from their phone number, which arrives the instant the call
    connects. Call `identify` for the sandbox demo caller:

    <CodeGroup>
    ```bash curl
    curl -X POST https://ringup.dev/api/tools/identify \
      -H "X-Caller-Number: +14155550142"
    ```

    ```javascript Node
    const res = await fetch("https://ringup.dev/api/tools/identify", {
      method: "POST",
      headers: { "X-Caller-Number": "+14155550142" },
    });
    const caller = await res.json();
    ```

    ```python Python
    import requests

    caller = requests.post(
        "https://ringup.dev/api/tools/identify",
        headers={"X-Caller-Number": "+14155550142"},
    ).json()
    ```
    </CodeGroup>

    ```json Response
    { "known": true, "name": "Alex", "card": { "brand": "VISA", "last_4": "5858" } }
    ```

    That is the identity layer: at the start of the call your agent greets a returning caller by
    name and can offer their saved card, with no card number spoken. See
    [Caller Identity](/concepts/identity).
  </Step>
  <Step title="Charge the saved card">
    A recognized caller with a saved card pays in one call: `charge` runs their card off-session and
    settles on the merchant's own processor. No card number is ever spoken, and nothing transfers.

    <CodeGroup>
    ```bash curl
    curl -X POST https://ringup.dev/api/tools/charge \
      -H "Content-Type: application/json" \
      -H "X-Caller-Number: +14155550142" \
      -H "X-Ringup-Merchant: tonys-pizza" \
      -d '{ "amount_cents": 2500 }'
    ```

    ```javascript Node
    const res = await fetch("https://ringup.dev/api/tools/charge", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "X-Caller-Number": "+14155550142",
        "X-Ringup-Merchant": "tonys-pizza",
      },
      body: JSON.stringify({ amount_cents: 2500 }),
    });
    const payment = await res.json();
    ```

    ```python Python
    import requests

    payment = requests.post(
        "https://ringup.dev/api/tools/charge",
        headers={
            "X-Caller-Number": "+14155550142",
            "X-Ringup-Merchant": "tonys-pizza",
        },
        json={"amount_cents": 2500},
    ).json()
    ```
    </CodeGroup>

    ```json Response
    { "status": "succeeded", "confirmation": "2PJJZY", "charged": { "brand": "VISA", "last_4": "5858", "amount_cents": 2500 } }
    ```

    That is the whole in-agent payment path: `identify` at the start, `charge` when the order is
    confirmed. Only `succeeded` moved money; read the `confirmation` back only then. Ringup applies
    the merchant's [payment policy](/concepts/payment-policy) server-side, so your agent never
    hard-codes when to charge. For a first-time caller with no saved card, [`send_link`](/concepts/send-link)
    texts a pay link instead. See [Charge](/concepts/charge).
  </Step>
  <Step title="Bring your own API key">
    The calls above run against the open demo sandbox with no key, so you can try them as-is. For
    your own merchants and callers you authenticate with your Ringup API key: create one in the
    dashboard and send it as `Authorization: Bearer <YOUR_RINGUP_KEY>`. See
    [Authentication](/authentication) for test versus live keys and where the key lives in each
    integration.
  </Step>
  <Step title="Add it to your agent">
    Both moves live inside your voice agent, and you integrate one of two ways. Every platform guide
    documents both as tabs:

    - **In your agent (API).** Keep the caller in your own agent and add the tools you just called
      (`identify`, then `charge` or [`send_link`](/concepts/send-link)). Nothing transfers, and no
      card data touches your agent. The flexible path.
    - **Hosted, from the dashboard.** Ringup takes the payment on a transfer or handoff, so you write
      no payment logic. Connect your platform once, pick the agents that take payments, rehearse on a
      test number, then go live. The easiest path.

    Pick your platform in the Integrations section: Vapi, Retell, ElevenLabs, Deepgram, LiveKit,
    OpenAI Realtime, Pipecat, and Twilio ConversationRelay.
  </Step>
</Steps>

<Note>
There is no per-merchant setup in either mode; merchants resolve from the dialed number on every
call, and each merchant activates for payments the moment it connects its own processor.
</Note>
