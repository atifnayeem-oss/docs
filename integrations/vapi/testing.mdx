Vapi tests against Ringup's shared sandbox, the same one every integration uses. The fixtures
live in one place, [Testing](/concepts/testing): the sandbox card (`4111 1111 1111 1111`), the
decline card, the test merchants, and the recognized test caller (`+14155550142`, saved Visa
ending 5858). This page covers the Vapi specifics for [call
transfer](/integrations/vapi/call-transfer), the recommended integration.

## Rehearse on a private line

When you add payment from the Ringup dashboard (see [Call
transfer](/integrations/vapi/call-transfer)), Ringup binds a test twin of your agent, with the
tools and prompt baked in, to a rehearsal number only you know about. Call it from a real
phone: you hear your actual agent take the order, the heads-up line, the transfer, and the full
payment flow, before anything you own changes.

Wiring by hand instead? Point the transfer tool's `server.url` at the keyless sandbox
(`https://ringup.dev/api/transfer-destination`) and call your agent. Two journeys to run:

1. **Recognized caller**: the test caller is greeted by name and pays on the saved Visa ending
   5858 with one yes.
2. **First-time caller**: call from a number the sandbox does not know; the payment line texts
   a secure link, you pay with `4111 1111 1111 1111`, and the call announces the payment the
   moment it lands.

Both should end with a confirmation code read back and a test-mode `checkout.succeeded`.

## Verify the wiring after go-live

Read the assistant back from the Vapi API and check three things:

- Both tools are attached: `identify` and `transfer_to_payment` appear in the assistant's
  `toolIds`, and the transfer tool is `async: true` with the Ringup `server.url`.
- The prompt block is present (identify at the greeting; transfer on order confirmation).
- The agent's phone number can transfer to SIP: Vapi-native and SIP-trunk numbers can; a
  number imported from a telephony provider cannot (the guide's warning covers the options).

<Note>
Vapi's dashboard does not always render apiRequest tool names; the Vapi API is the source of
truth for wiring.
</Note>

## Reading test results

Only a `succeeded` charge result with a `charge_id` proves a payment happened; an agent can
narrate success it did not have. And always validate with a **voice
call**, never text chat: transfers only exist on voice, so a chat smoke test proves nothing.
