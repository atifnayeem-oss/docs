Vapi tests against Ringup's shared sandbox, the same one every integration uses. The fixtures
live in one place, [Testing](/concepts/testing): the sandbox card (`4111 1111 1111 1111`), the
decline card, the test merchants, and the recognized test caller (`+14155550142`, saved Visa
ending 5858). This page covers the Vapi specifics: the rehearsal line and how to verify the
wiring.

## Rehearse on a private line

When you add payment from the Ringup dashboard (see
[Native handoff](/integrations/vapi/native)), Ringup binds a test twin of your agent, with the
go-live changes baked in, to a rehearsal number only you know about. Call it from a real
phone: you hear your actual agent, your actual prompts, and the full payment flow, before
anything you own changes. Wiring by hand instead? Build the same rig yourself; the Using the
API tab of the [native handoff guide](/integrations/vapi/native) walks it.

## Verify the wiring after go-live

Read the wiring back from the Vapi API: the squad structure, the payment and wrap-up
configuration, the handoff tool shape, and every transcriber model name (Vapi accepts typos
silently and they only fail at runtime).

<Note>
Vapi's dashboard does not render squad-scoped wiring or apiRequest tool names; the squad
canvas may show "No handoff tools configured" for working wiring. The Vapi API is the source
of truth.
</Note>

## Reading test results

Only a `succeeded` charge result with a payment id and confirmation code proves a payment
happened; an agent can narrate success it did not have. And validate squads with a **voice
call**: Vapi chat pipelines assemble handoffs differently, so a text-chat smoke test proves
nothing about voice.
