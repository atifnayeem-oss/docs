**Native handoff** keeps the caller in your agent's own voice from greeting to receipt: the
payment step runs inside your squad, so you write no payment prompt. The other option,
[call transfer](/integrations/vapi/call-transfer), standardizes one integration across several
platforms.

## Prerequisites

- A Vapi agent (or squad) you already run.
- Your Vapi API key (private key), and a Ringup [API key](/authentication).

## Set up

<Tabs>
<Tab title="Using the dashboard">

<Steps>
  <Step title="Connect Vapi, once for your account">
    In the Ringup dashboard, open **Integrations**, then **Connect Vapi**, and paste your Vapi
    API key (private key). Ringup uses it to list your agents and provision the payment objects,
    and never stores it. One connection covers every merchant you run.
  </Step>
  <Step title="Pick the agents that take payments">
    Select the agent that answers merchant calls (or the squad plus the member that confirms the
    order). Ringup assembles the payment step, the wrap-up step, the `ringup_*` tools, and the
    handoff tool around it, then shows the one prompt line to paste. No per-merchant step: the
    merchant resolves from the dialed number, and activates when it connects its processor.
  </Step>
  <Step title="Add your webhook endpoint">
    Under **Settings, then Webhooks**, add your HTTPS endpoint and copy the signing secret.
    Ringup posts a signed `checkout.completed` there for every payment. See
    [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Rehearse on the rehearsal number, then go live">
    The dashboard binds a test twin of your agent, with the go-live changes baked in, to a
    rehearsal number. Call it, order, and pay with the test card (Visa ending 5858): no real
    charge. When it reads a confirmation code back, click **Go live**. Ringup points your number
    at the squad, then makes exactly two changes to your assistant (appends the handoff tool and
    the one prompt line). **Roll back** any time restores it byte-for-byte.
  </Step>
</Steps>

</Tab>
<Tab title="Using the API">

Own every object in your own Vapi account: create the same pieces the dashboard registers,
assemble the squad, rehearse, and flip your number when ready.

<Steps>
  <Step title="Create the five ringup_* tools">
    Register `create_checkout`, `identify`, `charge`, `payment_status`, and `send_link` as
    static tools against Ringup's API. Caller and merchant resolve server-side from trusted
    headers, never from model-typed arguments. Call `identify` at the greeting, not at payment,
    so a returning caller is greeted by name
    ([Recognize at the start of the call](/concepts/identity)).
  </Step>
  <Step title="Create the payment step">
    An assistant that silently checks whether payment is needed for this purchase, then takes it,
    offers it, or says nothing. It inherits your agent's voice, model, and transcriber, and
    carries the five tools from the previous step.
  </Step>
  <Step title="Create the wrap-up step">
    Your own agent in wrap-up mode: its prompt verbatim with only the task replaced. It announces
    success, reads back the confirmation number, thanks the caller, and ends the call.
  </Step>
  <Step title="Assemble the squad">
    Create a new squad whose `members` array references your agent live (a reference, not a copy),
    then the payment step, then the wrap-up step. The payment step exits to wrap-up on its own,
    and the wrap-up ends the call; never hand back to your greeter. Your original agent and squad
    keep running untouched, and the new squad carries no traffic until a number points at it.
  </Step>
  <Step title="Create the handoff tool">
    A saved handoff tool (`handoff_to_Payment`) on the member that confirms the order. Attach it
    at go-live, not yet.
  </Step>
  <Step title="Add your webhook endpoint">
    In the Ringup dashboard, under **Settings, then Webhooks**, add your HTTPS endpoint and copy
    the signing secret. Your server verifies the signature and handles `checkout.completed`. See
    [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Test it, then go live">
    Point the tools at the keyless sandbox, `https://ringup.dev/api/tools`, build a rehearsal
    squad (twin, payment step, wrap-up step), and point a spare number at it. Call and order: the
    test caller (`+14155550142`) pays on the saved Visa ending 5858; a first-time caller is texted
    a link and enters `4111 1111 1111 1111`. Expect a confirmation code read back and a test-mode
    `checkout.completed`. To go live, point your phone number at the squad first, then make
    exactly two changes to your assistant: append the handoff tool to its `model.toolIds`, and
    append one prompt line (hand off once the order and name are confirmed). Keep your existing
    toolIds; a PATCH replaces the whole `model` object. **Roll back** by reversing exactly those
    two moves and pointing each number back where it was.
  </Step>
</Steps>

</Tab>
</Tabs>

<Warning>
Vapi's dashboard does not render squad-scoped wiring or apiRequest tool names; the squad canvas
may show "No handoff tools configured" for working wiring. Read the squad and assistant back from
the Vapi API to verify: it is the source of truth. Also, member `assistantOverrides` cannot
deliver tools on voice calls (toolIds are ignored, inline tools reach chat only), so the saved
assistant is the only wiring that works on voice.
</Warning>
