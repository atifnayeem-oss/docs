Add card-on-file payments to a Retell agent two ways. **Hosted** hands the call to Ringup at
payment time, so Ringup runs the whole payment and you write no payment logic. **In your agent**
keeps the caller in your agent the whole call: you call Ringup's tools to recognize the caller
and charge their saved card, and nothing transfers. A recognized caller never reads a card
aloud, so no card data touches your agent either way.

## Prerequisites

- A Retell agent you already run, with a phone number.
- Your Retell API key, and a Ringup [API key](/authentication).

## Set up

<Tabs>
<Tab title="Using the dashboard (hosted)">

Ringup takes payment on a transfer. Best when you want the payment moment handled for you.

<Steps>
  <Step title="Connect Retell, once for your account">
    In the Ringup dashboard, open **Integrations**, then **Connect Retell**, and paste your Retell
    API key. Ringup uses it to list your agents and provision the payment objects, and never
    stores it. One connection covers every merchant you run.
  </Step>
  <Step title="Pick the agents that take payments">
    Select the agents that answer merchant calls. Ringup registers `identify` (called at the
    greeting), `create_checkout`, and the `transfer_to_payment` tool on each; paste the one prompt
    line the dashboard shows. No per-merchant step: the merchant resolves from the dialed number,
    and activates when it connects its processor.
  </Step>
  <Step title="Add your webhook endpoint">
    Under **Settings, then Webhooks**, add your HTTPS endpoint and copy the signing secret. Ringup
    posts a signed `checkout.completed` there for every payment. See [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Rehearse on the rehearsal number, then go live">
    The dashboard binds a test twin of your agent to a rehearsal number. Call it, order, and pay
    with the test card (Visa ending 5858): no real charge. When it reads a confirmation code back,
    click **Go live**.
  </Step>
</Steps>

<Warning>
The hosted transfer needs a number Retell can move a call from: numbers bought through Retell and
numbers imported over an elastic SIP trunk both work (for a trunk number, enable call transfer /
SIP REFER at your provider). If your number cannot transfer, use the **In your agent** tab, which
never transfers.
</Warning>

</Tab>
<Tab title="In your agent (API)">

Keep the caller in your agent and add three Retell custom functions pointed at Ringup, authed
with your Ringup key. The caller's number rides the `X-Caller-Number` header Retell fills, so the
model never types identity.

<Steps>
  <Step title="Recognize the caller at the greeting">
    Register `identify` and call it at the greeting, not at payment. It takes no arguments; for a
    returning caller it returns their name and saved card, so your agent greets them by name and
    knows a saved card is on file. See [Recognize at the start of the call](/concepts/identity).

    ```jsonc
    {
      "type": "custom",
      "name": "identify",
      "description": "Recognize the caller at the greeting. Returns their name and whether a saved card is on file.",
      "url": "https://api.ringup.dev/v1/identify",
      "method": "POST",
      "args_at_root": true,
      "headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>", "X-Caller-Number": "{{user_number}}" },
      "parameters": {}
    }
    ```
  </Step>
  <Step title="Charge the saved card at payment">
    Register `charge`. When the order is confirmed and `identify` returned a saved card, your agent
    calls it with the order total; Ringup charges the saved card off-session on the merchant's own
    processor and returns the confirmation to read back. It respects the merchant's payment policy
    server-side, so on an unconnected merchant it is a silent no-op.

    ```jsonc
    {
      "type": "custom",
      "name": "charge",
      "description": "Charge the recognized caller's saved card for the confirmed order. Returns the confirmation code to read back.",
      "url": "https://api.ringup.dev/v1/charge",
      "method": "POST",
      "args_at_root": true,
      "headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>", "X-Caller-Number": "{{user_number}}" },
      "parameters": {
        "amount_cents": { "type": "number", "description": "Order total in cents" },
        "order_id":     { "type": "string", "description": "Your order id, if you have one" }
      },
      "speak_during_execution": true,
      "execution_message_type": "static_text",
      "execution_message_description": "One moment while I put that through."
    }
    ```
  </Step>
  <Step title="Text a link for a first-time caller">
    Register `send_link` for a caller with no saved card (or a declined charge). Ringup texts a
    secure pay link and, when they pay, pushes the confirmation back into the live call (or sends
    it by SMS if the call has ended).

    ```jsonc
    {
      "type": "custom",
      "name": "send_link",
      "description": "Text the caller a secure pay link for the order. Use when there is no saved card on file.",
      "url": "https://api.ringup.dev/v1/send_link",
      "method": "POST",
      "args_at_root": true,
      "headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>", "X-Caller-Number": "{{user_number}}" },
      "parameters": { "amount_cents": { "type": "number", "description": "Order total in cents" } }
    }
    ```

    Then add one line to your prompt: *when the caller has confirmed the order, if identify
    returned a saved card call `charge` with the order total and read back the confirmation code;
    otherwise call `send_link` and tell them to check their text to pay.*
  </Step>
  <Step title="Add your webhook endpoint">
    In the Ringup dashboard under **Settings, then Webhooks**, add your HTTPS endpoint and copy the
    signing secret. Your server verifies the signature and handles `checkout.completed`. See
    [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Test it, then go live">
    Point the functions' `url` at the keyless sandbox, `https://ringup.dev/api/tools` (or keep the
    prod URLs with your `rk_test_` key). Call your agent and order: the test caller
    (`+14155550142`) is recognized and pays on the saved Visa ending 5858; a first-time caller is
    texted a link and enters `4111 1111 1111 1111`. Expect the confirmation read back and a
    test-mode `checkout.completed`. Swap to the live URLs to go live.
  </Step>
</Steps>

</Tab>
