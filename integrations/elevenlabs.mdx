Add card-on-file payments to an ElevenLabs agent two ways. **Hosted** transfers the call to Ringup
at payment time, so Ringup runs the whole payment and you write no payment logic. **In your agent**
keeps the caller in your agent the whole call: you add three webhook tools that recognize the caller
and charge their saved card, and nothing transfers. A recognized caller never reads a card aloud, so
no card data touches your agent either way.

## Prerequisites

- An ElevenLabs agent you already run, with a phone number.
- Your ElevenLabs API key, and a Ringup [API key](/authentication).

## Set up

<Tabs>
<Tab title="Using Dashboard">

Ringup takes payment on a transfer. Best when you want the payment moment handled for you.

<Warning>
The hosted transfer needs the agent's number attached over a **SIP trunk**. ElevenLabs allows a
SIP-URI transfer destination only with `sip_refer`, and rejects that combination on a native-Twilio
number, so an agent on a native-Twilio number cannot reach the payment line by transfer. If your
number cannot transfer, use the **Using Tools** tab, which never transfers.
</Warning>

<Warning>
**Preview.** The transfer transport is live: an agent on a SIP-trunk number can refer the call to
the payment line. The piece that matches the transferred call to the order you registered by the
caller's number is still in build. Confirm availability in your Ringup dashboard before relying on
the hosted path in production. The **Using Tools** tab is not affected and works today.
</Warning>

<Steps>
  <Step title="Connect ElevenLabs, once for your account">
    In the Ringup dashboard, open **Integrations**, then **Connect ElevenLabs**, and paste your
    ElevenLabs API key. Ringup uses it to list your agents and provision the payment objects, and
    never stores it. One connection covers every merchant you run.
  </Step>
  <Step title="Pick the agents that take payments">
    Select the agents that answer merchant calls. Ringup registers `identify` (called at the
    greeting) and `create_checkout` on each and pins your payment address in the agent's transfer
    rule; paste the one prompt line the dashboard shows. No per-merchant step: the merchant resolves
    from the dialed number, and activates when it connects its processor.
  </Step>
  <Step title="Add your webhook endpoint">
    Under **Settings, then Webhooks**, add your HTTPS endpoint and copy the signing secret. Ringup
    posts a signed `checkout.succeeded` there for every payment. See [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Rehearse on the rehearsal number, then go live">
    The dashboard binds a test twin of your agent to a rehearsal number. Call it, order, and pay
    with the test card (Visa ending 5858): no real charge. When it reads a confirmation code back,
    click **Go live**.
  </Step>
</Steps>

</Tab>
<Tab title="Using Tools">

Keep the caller in your agent and add four ElevenLabs **webhook tools** pointed at Ringup, authed
with your Ringup key. `caller_phone` is pinned to the `system__caller_id` dynamic variable and
`merchant_id` to a per-merchant one, so the model never types either identity, and nothing
transfers.

<Steps>
  <Step title="Recognize the caller at the greeting">
    Add a webhook tool `identify` and call it at the greeting, not at payment. For a returning
    caller it returns their name, so your agent greets them by name and skips questions it already
    has answers to. An unknown caller comes back `known: false`, which is an ordinary first call.
    See [Recognize at the start of the call](/concepts/identity).

    ```jsonc
    {
      "type": "webhook",
      "name": "identify",
      "description": "Recognize the caller at the greeting. Returns their name if Ringup knows them.",
      "api_schema": {
        "url": "https://api.ringup.dev/v1/identify",
        "method": "POST",
        "request_headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>" },
        "request_body_schema": {
          "type": "object",
          "properties": {
            "caller_phone": { "type": "string", "dynamic_variable": "system__caller_id" }
          },
          "required": ["caller_phone"]
        }
      }
    }
    ```
  </Step>
  <Step title="Open a Checkout Session when the order is confirmed">
    Add a webhook tool `create_checkout`. It resolves the total, applies the merchant's payment
    policy, and returns `cards`: the caller's cards chargeable at this merchant. If
    `payment_required` is `none`, say nothing about payment and finish the call.

    ```jsonc
    {
      "type": "webhook",
      "name": "create_checkout",
      "description": "Open a payment session for the confirmed order. Returns whether payment is required and which saved cards can be used.",
      "api_schema": {
        "url": "https://api.ringup.dev/v1/create_checkout",
        "method": "POST",
        "request_headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>" },
        "request_body_schema": {
          "type": "object",
          "properties": {
            "merchant_id":  { "type": "string", "dynamic_variable": "merchant_id" },
            "caller_phone": { "type": "string", "dynamic_variable": "system__caller_id" },
            "order_id":     { "type": "string", "description": "The merchant's order id, if you have one. Ringup reads the total from it." },
            "amount_cents": { "type": "number", "description": "Order total in cents. Required only when there is no order_id." }
          },
          "required": ["merchant_id", "caller_phone"]
        }
      }
    }
    ```
  </Step>
  <Step title="Charge the card the caller picked">
    Add a webhook tool `charge`. Read one card from `cards` back by brand and last four with the
    amount, and call `charge` only after the caller agrees, with that card's `instrument_id`.

    ```jsonc
    {
      "type": "webhook",
      "name": "charge",
      "description": "Charge the card the caller agreed to. Use the checkout_session_id and the instrument_id of the card you read back.",
      "api_schema": {
        "url": "https://api.ringup.dev/v1/charge",
        "method": "POST",
        "request_headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>" },
        "request_body_schema": {
          "type": "object",
          "properties": {
            "checkout_session_id": { "type": "string", "description": "From create_checkout" },
            "instrument_id":       { "type": "string", "description": "The card the caller agreed to, from the session's cards" }
          },
          "required": ["checkout_session_id", "instrument_id"]
        }
      }
    }
    ```
  </Step>
  <Step title="Text a link when there is no card to offer">
    Add a webhook tool `send_link` for a session whose `cards` is empty (or after a declined
    charge). Ringup texts a secure pay link and, when they pay, pushes the confirmation back into
    the live call (or sends it by SMS if the call has ended).

    ```jsonc
    {
      "type": "webhook",
      "name": "send_link",
      "description": "Text the caller a secure pay link for this session. Use when the session returned no cards.",
      "api_schema": {
        "url": "https://api.ringup.dev/v1/send_link",
        "method": "POST",
        "request_headers": { "Authorization": "Bearer <YOUR_RINGUP_KEY>" },
        "request_body_schema": {
          "type": "object",
          "properties": {
            "checkout_session_id": { "type": "string", "description": "From create_checkout" }
          },
          "required": ["checkout_session_id"]
        }
      }
    }
    ```

    Then add one line to your prompt: *when the caller has confirmed the order, call
    `create_checkout`; if it returns cards, read one back and call `charge` after they agree, then
    read back the order id as the confirmation; if it returns none, call `send_link` and tell them
    to check their text to pay.*
  </Step>
  <Step title="Add your webhook endpoint">
    In the Ringup dashboard under **Settings, then Webhooks**, add your HTTPS endpoint and copy the
    signing secret. Your server verifies the signature and handles `checkout.succeeded`. See
    [Webhooks](/concepts/webhooks).
  </Step>
  <Step title="Test it, then go live">
    Point the tools' `url` at the keyless sandbox, `https://ringup.dev/api/tools` (or keep the prod
    URLs with your `rk_test_` key). Call your agent and order: the test caller (`+14155550142`) is
    recognized and pays on the saved Visa ending 5858; a first-time caller is texted a link and
    enters `4111 1111 1111 1111`. Expect the confirmation read back and a test-mode
    `checkout.succeeded`. Swap to the live URLs to go live.
  </Step>
</Steps>

</Tab>
